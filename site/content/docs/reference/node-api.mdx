## The `Entl` class

### `Entl`

An open entl database. Heavy ops are unary (off-thread in async hosts).

```ts
class Entl {
  /** Open (or create) the .duckdb at `dbPath` and apply the schema. */
  constructor(dbPath: string)
  /** Load git history from `repoPath` (one-way, incremental). */
  loadGit(repoPath: string): Promise<GitStats>
  /** Load GitHub data (events/PRs/issues/Actions). Needs a token. */
  loadGithub(repoPath: string): Promise<GithubStats>
  /** Run a SQL query; JSON rows back. */
  query(sql: string): Promise<string>
  /**
   * Run a SQL query; the result as one Arrow IPC stream (schema + all batches) —
   * the dataframe on-ramp (pyarrow → pandas/polars, apache-arrow JS, red-arrow).
   */
  queryArrow(sql: string): Promise<Buffer>
  /** Pull `repoPath` and sync it into a target store, in one call. */
  sink(repoPath: string, options: SinkOptions): Promise<SinkStats>
  /** Read a store back into canonical rows (JSON; oids hex, timestamps RFC3339). */
  extract(options: ExtractOptions): Promise<string>
  /** Stream the change batches from one pull (the stream plane). */
  changes(repoPath: string, options?: ChangesOptions | undefined | null): Changes
  /** Backfill this store into a driver target: stream {sql, params} for the host to execute. */
  driverPlan(options?: DriverPlanOptions | undefined | null): DriverPlan
  /** Reconstruct a git repo from a store (needs objects: true at sink time). Returns commits rebuilt. */
  rebuild(options: RebuildOptions): Promise<number>
  /**
   * Watch `repoPath`: on a background thread, every `intervalSecs`, `git fetch`
   * + load git + load GitHub into the DB, then call `onSync(stats)` so the JS
   * side can mirror (e.g. into PGlite, whose `live` drives realtime). Returns a
   * handle; call `.stop()` to end it. The first cycle runs immediately.
   */
  watch(repoPath: string, intervalSecs: number, onSync: (stats: SyncStats) => void): WatchHandle
}
```

## Types

### `ChangeBatch`

One change-stream batch. The rows stay columnar (Arrow): `ipc` yields them as one Arrow IPC stream (a single RecordBatch, schema included), and the Python class additionally speaks the Arrow PyCapsule protocol for zero-copy import (`pa.record_batch(batch)`). Values are native Arrow types — oids are Binary, timestamps Timestamp(µs) — unlike query()/extract()'s canonical JSON (hex oids, RFC3339). No per-row `_op`: `op`/`table` ride this envelope.

```ts
class ChangeBatch {
  get table(): string
  get op(): string
  /** The rows as one Arrow IPC stream — decode with `tableFromIPC` (apache-arrow). */
  get ipc(): Buffer
}
```
